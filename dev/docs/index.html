JxoOS Developer Documentation

📦 Creating a .jxo App Package

To create a .jxo file (JxoOS app package), follow this folder structure:

Example/
├── assets/
│   └── app/
│       ├── index.html
│       ├── styles.css
│       └── main.js
│   └── images/
│       ├── main.png
│       └── other.png
├── config.jxo

config.jxo Format

App_name: Example
App_image_path: assets/images/main.png
App_main: assets/app/index.html
App_css(s): assets/app/styles.css
App_js(s): assets/app/main.js
Other_files: assets/app/data.json
Dev_only: true

Note: Multiple CSS and JS files can be separated by commas.

💻 Creating a .jxiso OS Package

To create a .jxiso file (JxoOS operating system package), use this structure:

ExampleOS/
├── assets/
│   ├── css/
│   ├── js/
│   └── html/
│       ├── main.html
│       └── boot.html
├── taskbar.png
├── icon.png
└── os.jxo

os.jxo Format

OS_name: Example
OS_icon_path: icon.png
Is_default: true
Kiosk_mode: false

Kiosk_mode: Prevents switching OS via Developer Mode when set to true.

📁 Packaging Your Files

Use the packaging tool at:

https://jxoj.github.io/Jxo-OS/dev

🛠️ Developer Mode Info

  • Access Dev Mode by powering down the OS and clicking the Developer Mode button.
  • Dev Mode enables uploading .jxo and .jxiso packages directly into the OS.
  • Apps marked Dev_only: true will only appear in Developer Mode.
