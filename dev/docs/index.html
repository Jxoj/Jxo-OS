JxoOS Developer API Documentation

Endpoints

  • POST /dev/package/app

    Uploads a zipped folder of your app structure and returns a packaged .jxo file.

    Form Field: folderZip – a ZIP file containing your app folder with the required structure and a config.jxo file.

  • POST /dev/package/os

    Uploads a zipped folder of your OS structure and returns a packaged .jxiso file.

    Form Field: folderZip – a ZIP file containing your OS folder with the required structure and an os.jxo file.

Folder Structures

.jxo App Package

Example/
├── assets/
│   └── app/
│       ├── index.html
│       ├── styles.css
│       └── main.js
├── images/
│   ├── main.png
│   └── example.png
└── config.jxo

config.jxo sample content:

App_name: Example
App_image_path: assets/images/main.png
App_main: assets/app/index.html
App_css(s): assets/app/styles.css,assets/app/morecss.css
App_js(s): assets/app/main.js,assets/app/morejs.js
Other_files: assets/app/main.json,assets/app/main.txt
Dev_only: true

.jxiso OS Package

ExampleOS/
├── assets/
│   ├── css/
│   ├── js/
│   └── html/
│       ├── main.html
│       └── boot.html
├── taskbar.png
├── icon.png
└── os.jxo

os.jxo sample content:

OS_name: Example
OS_icon_path: icon.png
Is_default: true
Kiosk_mode: false

Accessing OS Functions

Even if another OS is loaded (overlaid), you can still call its functions using:

jxoos.myFunction(arg1, arg2);
